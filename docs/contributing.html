Contributing to OpenWatch

Thank you for your interest in contributing! Here's how you can help.

Getting Started

  1. Fork the repository on GitHub
  2. Clone your fork: 
    git clone https://github.com/YOUR_USERNAME/openwatch.git
cd openwatch
  3. Install dependencies: 
    npm install
  4. Start development server: 
    npm run dev

Development Guidelines

  • Code Style: Follow the existing code patterns. Use ESLint and Prettier.
  • Commits: Write clear, descriptive commit messages.
  • Tests: Add tests for new features when applicable.
  • Documentation: Update documentation for any changes.
  • i18n: Add translations for all new strings in all 5 supported languages.

Project Structure

openwatch/
├── app/
│   ├── components/     # Vue components
│   │   ├── log/        # Log Viewer components
│   │   ├── profile/    # Profile Viewer components
│   │   ├── trace/      # Trace Viewer components
│   │   └── network/    # Network Inspector components
│   ├── composables/    # Vue composables (stores, parsers)
│   ├── locales/        # i18n translation files
│   ├── pages/          # Nuxt pages
│   └── types/          # TypeScript type definitions
├── docs/               # GitHub Pages documentation
└── public/             # Static assets

Areas for Contribution

  • New Tools: Metrics Viewer, Heap Snapshot, Lighthouse Report
  • Features: Export functionality, additional parsers, themes
  • Translations: Improve or add new language support
  • Bug Fixes: Check the Issues tab for known bugs
  • Documentation: Improve guides and examples

Submitting Changes

  1. Create a new branch: git checkout -b feature/my-feature
  2. Make your changes
  3. Test your changes locally
  4. Commit: git commit -m "Add: my new feature"
  5. Push: git push origin feature/my-feature
  6. Open a Pull Request on GitHub

Need Help?

Feel free to open an issue on GitHub if you have questions or need guidance.
